# Техническое задание на разработку мобильного приложения

## 1. Общие положения

### 1.1. Наименование проекта
Мобильное приложение "TaskManager Pro" для iOS и Android.

### 1.2. Цель проекта
Создание кроссплатформенного мобильного приложения для управления задачами с функциями:
- Создание и редактирование задач
- Назначение сроков выполнения
- Категоризация задач по проектам
- Уведомления о дедлайнах
- Синхронизация между устройствами

## 2. Требования к функциональности

### 2.1. Основные модули приложения

#### 2.1.1. Модуль аутентификации
- Регистрация по email и паролю
- Вход в систему
- Восстановление пароля
- Выход из системы

#### 2.1.2. Модуль задач
- Создание новой задачи (название, описание, срок, приоритет)
- Редактирование существующих задач
- Удаление задач
- Отметка о выполнении
- Фильтрация по статусу (все, активные, завершенные)

#### 2.1.3. Модуль проектов
- Создание проектов
- Добавление задач в проекты
- Просмотр статистики по проекту
- Архивация завершенных проектов

### 2.2. Дополнительные функции
- Push-уведомления о приближении дедлайна
- Темная/светлая тема оформления
- Экспорт задач в PDF
- Интеграция с Google Календарём

## 3. Технические требования

### 3.1. Платформы
- **iOS**: версия 14.0 и выше
- **Android**: версия 8.0 (API 26) и выше

### 3.2. Архитектура приложения


### 3.3. Стек технологий
- **Frontend**: React Native 0.70
- **Backend**: Node.js 18 + Express
- **База данных**: PostgreSQL 15
- **Хранение файлов**: Amazon S3
- **Push-уведомления**: Firebase Cloud Messaging

### 3.4. API
- REST API с аутентификацией по JWT токену
- Документация в формате OpenAPI 3.0
- Rate limiting: 100 запросов в минуту на пользователя

## 4. Требования к интерфейсу

### 4.1. Общие принципы
- Material Design для Android
- Human Interface Guidelines для iOS
- Поддержка жестов (свайпы, долгое нажатие)
- Адаптивность под разные размеры экранов

### 4.2. Основные экраны
1. **Экран входа/регистрации**
2. **Главный экран со списком задач**
3. **Экран создания/редактирования задачи**

## 5. Требования к безопасности

### 5.1. Защита данных
- Шифрование паролей (bcrypt)
- HTTPS для всех сетевых запросов
- Валидация входных данных
- Защита от SQL-инъекций

### 5.2. Конфиденциальность
- Политика конфиденциальности согласно GDPR
- Возможность удаления аккаунта и всех данных
- Шифрование локальной базы данных
